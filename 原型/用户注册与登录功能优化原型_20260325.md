### 1.2 现有注册流程问题

```
现有流程：
用户输入 username/email/password → 直接注册 → 返回 token

问题点：
1. 缺少登录功能，用户注册后无法重新登录
2. 没有 Token 刷新机制，24小时后 token 过期需重新注册
3. 没有第三方登录（微信/Google），用户体验差
4. 没有密码找回功能，忘记密码只能重新注册
5. 没有用户头像、昵称等个人信息
6. 缺少登录安全保护（失败次数限制、验证码）
```

## 三、功能详细设计

### 3.1 用户注册（优化）

**页面路径：** `/register`

**交互流程：**

```
┌─────────────────────────────────────────────────────────────┐
│                    📝 创建账号                               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐                    │
│  │ 📧 邮箱 │  │ 👤 用户名│  │ 🔒 密码 │                    │
│  └─────────┘  └─────────┘  └─────────┘                    │
│                                                             │
│  邮箱：  [user@example.com        ] ⚠️ 用于找回密码           │
│  用户名：[漫旅创作者              ] ⚠️ 2-20字符，字母/数字/中文 │
│  密码：  [••••••••              ] ⚠️ 至少8位，字母+数字       │
│  确认密码：[••••••••              ]                        │
│                                                             │
│  ☐ 我已阅读并同意《用户协议》和《隐私政策》                    │
│                                                             │
│           [━━━━━━━━━━ 创建账号 ━━━━━━━━━━]                   │
│                                                             │
│  ─────────────────────────────────────────────────────      │
│  已有账号？ [登录]                                           │
└─────────────────────────────────────────────────────────────┘
```

**字段规则：**

| 字段 | 规则 | 错误提示 |
|------|------|----------|
| 邮箱 | Email 格式，唯一性校验 | "邮箱格式错误" / "该邮箱已被注册" |
| 用户名 | 2-20字符，支持中英文/数字/下划线 | "用户名已被占用" / "格式不符合要求" |
| 密码 | 最少8位，建议包含字母+数字 | "密码强度不足" |
| 确认密码 | 与密码一致 | "两次输入密码不一致" |

**业务逻辑：**

```python
# 注册成功后的处理
1. 密码加密存储（bcrypt）
2. 创建用户记录，获取 user_id
3. 生成 JWT Access Token（24小时）
4. 生成 Refresh Token（7天）
5. 返回 { user_id, access_token, refresh_token, expires_in }
```

### 3.3 Token 管理（新增）

**Token 结构：**

| 类型 | 有效期 | 用途 |
|------|--------|------|
| Access Token | 24小时 | API 鉴权 |
| Refresh Token | 7天 | 刷新 Access Token |

**Token 刷新流程：**

```
┌─────────────────────────────────────────────────────────────┐
│                    Token 自动刷新                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  客户端存储：                                               │
│  ├── access_token: "eyJhbG..."                             │
│  ├── refresh_token: "eyJhbG..."                            │
│  └── expires_at: 1742870400  (2026-03-25 12:00:00)         │
│                                                             │
│  刷新时机：                                                 │
│  ├── Access Token 剩余有效期 < 5分钟 → 自动刷新             │
│  ├── 收到 401 Unauthorized → 尝试刷新                      │
│  └── 刷新失败 → 跳转登录页                                  │
│                                                             │
│  刷新接口：                                                 │
│  POST /api/auth/refresh                                    │
│  Body: { "refresh_token": "xxx" }                          │
│  Response: { "access_token": "xxx", "expires_in": 86400 } │
└─────────────────────────────────────────────────────────────┘
```

**退出登录：**

```
POST /api/auth/logout
Header: Authorization: Bearer <access_token>
Body: { "refresh_token": "xxx" }

# 服务端处理：
1. 将 refresh_token 加入黑名单
2. 清除客户端本地存储
3. 返回成功
```

### 4.2 登录接口（新增）

```yaml
POST /api/auth/login
Content-Type: application/json

Request:
{
  "email": "user@example.com",
  "password": "correct_password"
}

Response (200):
{
  "user_id": 123,
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "refresh_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "expires_in": 86400,        # access_token 有效期（秒）
  "token_type": "Bearer"
}

Error Responses:
- 400: { "detail": "请求格式错误" }
- 401: { "detail": "邮箱或密码错误" }
- 429: { "detail": "登录失败次数过多，请15分钟后再试" }
```

```sql
ALTER TABLE users ADD COLUMN is_vip BOOLEAN DEFAULT FALSE;
ALTER TABLE users ADD COLUMN vip_expires_at DATETIME;
ALTER TABLE users ADD COLUMN last_login_at DATETIME;
ALTER TABLE users ADD COLUMN email_verified BOOLEAN DEFAULT FALSE;

-- Token 黑名单表
CREATE TABLE token_blacklist (
    id INT PRIMARY KEY AUTO_INCREMENT,
    token_jti VARCHAR(100) UNIQUE NOT NULL,  -- JWT ID
    revoked_at DATETIME NOT NULL,
    expires_at DATETIME NOT NULL,
    INDEX idx_token_jti (token_jti),
    INDEX idx_expires_at (expires_at)
);

-- 登录日志表
CREATE TABLE login_logs (
    id INT PRIMARY KEY AUTO_INCREMENT,
    user_id INT,
    ip_address VARCHAR(45),
    user_agent VARCHAR(500),
    login_status VARCHAR(20),     -- success/failed
    fail_reason VARCHAR(100),
    created_at DATETIME,
    FOREIGN KEY (user_id) REFERENCES users(id)
);
```

### 5.2 Redis 缓存设计

```python
# 登录失败计数（防暴力破解）
Key: "login_fail:{ip}:{email}"
Value: count
TTL: 300秒（5分钟）

# 账户锁定标记
Key: "account_lock:{user_id}"
Value: locked_until (timestamp)
TTL: 86400秒（24小时）

# Refresh Token 存储
Key: "refresh_token:{user_id}:{jti}"
Value: token_data
TTL: 604800秒（7天）
```

## 六、UI 交互细节

### 6.1 注册页动画

```
输入校验反馈：
├── 邮箱格式错误 → 边框变红 + 抖动动画 + 下方提示
├── 用户名可用 → 边框变绿 + ✓ 图标
├── 密码强度不足 → 进度条变红 + 提示"建议：字母+数字组合"
└── 提交中 → 按钮显示加载动画，禁用重复点击
```

### 6.2 登录错误处理

```
错误场景：
├── 密码错误 → 输入框抖动 + "邮箱或密码错误"（不明确指出哪个错误）
├── 账号锁定 → 显示剩余锁定时间 + "账号安全保护中"
├── 网络错误 → 顶部 toast + 自动重试按钮
└── 登录成功 → 按钮变绿 + 跳转到首页/创作页面
```

### 6.3 Token 刷新提示

```
刷新成功 → 无感知，静默更新
刷新失败 → 弹窗"登录已过期，请重新登录" + 跳转登录页
```
