# 角色一致性功能原型

> 日期：2026-03-23 | 版本：v1.0 | 状态：初稿
> 参考：竞品分析报告_20260322.md

---

## 一、产品概述

### 1.1 核心问题

当前AI视频/动态漫工具（Runway/Pika/海螺/即梦）生成的角色存在"每张图不一样"的问题，用户无法确保同一角色在不同镜头中保持视觉一致性。

### 1.2 解决方案

构建**角色库系统**，通过角色ID + LoRA特征注入，确保角色跨镜头一致性达到 **≥85%**。

---

## 二、功能架构

### 2.1 角色库管理

```
角色库（角色卡片集合）
├── 角色卡片 A（主角：小明）
│   ├── 基础形象（参考图 3-5张）
│   ├── 角色ID（全局唯一）
│   ├── 角色名称/描述
│   ├── 绑定LoRA模型
│   └── 关联项目列表
├── 角色卡片 B（女主：小红）
└── ...
```

### 2.2 角色卡片数据结构

| 字段 | 类型 | 说明 |
|------|------|------|
| role_id | UUID | 全局唯一标识 |
| role_name | String | 角色名称 |
| reference_images | Array[URL] | 参考图（3-5张，多角度） |
| character_features | Object | 特征向量（系统提取） |
| lora_model_path | String | 绑定的LoRA模型路径 |
| style_prompt | String | 角色风格描述词 |
| project_ids | Array[String] | 关联项目ID列表 |
| created_at | DateTime | 创建时间 |
| updated_at | DateTime | 更新时间 |

## 三、核心功能流程

### 3.1 角色创建流程（新建角色）

```
Step 1：用户上传参考图
        ↓
Step 2：系统提取角色特征
        （面部结构、服装风格、发色、瞳色等）
        ↓
Step 3：生成角色ID（UUID）
        ↓
Step 4：用户填写角色名称/描述（可选）
        ↓
Step 5：系统自动生成风格Prompt模板
        ↓
Step 6：角色卡片创建完成，进入角色库
```

### 3.2 角色ID跨镜头注入流程

```
镜头生成请求
    ↓
携带角色ID + 当前镜头Prompt
    ↓
角色服务查找角色卡片
    ↓
提取 character_features + lora_model_path
    ↓
将特征注入到生成参数
    ↓
调用图生图/文生图模型
    ↓
输出带角色一致性的镜头画面
```

## 四、角色库管理功能

### 4.1 角色库列表页

**功能描述：**
- 展示当前项目/全局角色库
- 支持角色搜索（按名称/标签）
- 支持角色分组（主角/配角/群演）

**交互元素：**
- 角色缩略图卡片（显示角色名、参考图数量）
- 新建角色按钮
- 批量导入角色
- 角色详情入口

### 4.2 角色详情页

**功能描述：**
- 查看/编辑角色信息
- 管理参考图
- 绑定/训练LoRA
- 查看使用记录

**交互元素：**
- 参考图画廊（可增删）
- 角色特征标签（自动提取+手动添加）
- LoRA训练按钮（触发后端训练任务）
- "应用到所有镜头"按钮

### 4.3 LoRA管理

**功能描述：**
- 角色可绑定预训练LoRA
- 支持从角色特征自动生成轻量LoRA
- LoRA模型列表管理

**交互元素：**
- 绑定现有LoRA下拉框
- 触发LoRA训练按钮
- 训练进度条
- LoRA强度滑块（0.0-1.0）

## 五、角色ID一致性保障机制

### 5.1 一致性计算公式

```
镜头一致性得分 = α × 面部相似度 + β × 服装相似度 + γ × 风格一致性

其中：
α + β + γ = 1.0
默认权重：α=0.5, β=0.3, γ=0.2
```

### 5.2 一致性保障策略

| 场景 | 策略 |
|------|------|
| 新镜头生成 | 强制注入角色ID + 角色特征 |
| LoRA可用时 | 启用LoRA增强一致性 |
| LoRA不可用时 | 使用character_features特征注入 |
| 一致性得分 < 85% | 提示用户优化或重新生成 |

### 5.3 异常处理

- **参考图不足**：提示用户补充至少3张不同角度参考图
- **角色特征提取失败**：降级为纯Prompt方式，提示一致性可能下降
- **LoRA训练失败**：记录错误，允许用户重试或跳过

## 六、与其他模块的交互

### 6.1 与分镜模块

- 分镜面板中每个镜头可选择绑定角色
- 镜头生成时自动携带角色ID
- 分镜预览支持角色信息叠加显示

### 6.2 与图生图模块

- 角色ID作为必传参数
- 支持"仅保持角色一致性"和"保持一致+动作变化"两种模式

### 6.3 与导出模块

- 导出报告包含角色一致性评分
- 便于用户了解成片质量

## 八、技术实现要点

### 8.1 角色特征提取

- 使用 CLIP/VIT 或自研特征提取模型
- 提取后存储为向量，支持向量检索

### 8.2 LoRA训练

- 使用开源LoRA训练框架（如Kohya）
- 训练触发方式：手动触发 + 自动触发（角色使用次数≥10时）
- 训练优先级：主角 > 配角 > 群演

### 8.3 一致性检测

- 每批次生成后自动计算一致性得分
- 一致性得分低于阈值时自动提示
