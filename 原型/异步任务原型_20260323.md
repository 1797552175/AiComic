> 日期：2026-03-23 | 版本：v1.0 | 状态：初稿
> 参考：竞品分析报告_20260322.md

## 一、产品概述

### 1.1 核心问题

AI动态漫生成是计算密集型任务，单个镜头生成需10-60秒，整部作品生成可能需要数分钟。传统同步请求模式会导致：
- 前端请求超时（HTTP 504）
- 用户无法感知进度，以为系统卡死
- 并发控制困难，容易击垮服务端

### 1.2 解决方案

采用 **Celery 异步任务队列 + WebSocket 实时推送** 架构，让用户实时感知生成进度，任务可取消、可重试，服务端稳定可靠。

## 三、核心功能设计

### 3.1 任务创建

**接口：POST /api/v1/tasks**

**请求体：**
```json
{
  "task_type": "project_generate",
  "params": {
    "project_id": "proj_123",
    "script": "...",
    "characters": ["role_abc", "role_def"],
    "style": "日系动漫"
  },
  "callback_url": "https://your-app.com/webhook/task_done",
  "priority": "normal"
}
```

**响应体：**
```json
{
  "task_id": "task_7a3b9c2d",
  "status": "pending",
  "created_at": "2026-03-23T16:00:00Z",
  "websocket_url": "wss://api.example.com/ws/tasks/task_7a3b9c2d"
}
```

### 3.2 任务状态机

```
                    ┌──────────────┐
                    │    pending   │ ← 任务创建，等待执行
                    └──────┬───────┘
                           │ Worker获取任务
                           ▼
                    ┌──────────────┐
              ┌─────│   queued     │ ← 任务在队列中
              │     └──────┬───────┘
              │            │ Worker开始执行
              ▼            ▼
       ┌────────────┐ ┌──────────────┐
       │ cancelled  │ │   running    │ ← 任务执行中
       │ (用户取消)  │ └──────┬───────┘
       └────────────┘        │
                             │ 执行完成/失败
                             ▼
                    ┌──────────────┐
                    │   success    │ / │   failed
                    └──────────────┘
```

### 3.3 任务类型

| 任务类型 | 描述 | 预计时长 | 资源需求 |
|----------|------|----------|----------|
| `shot_generate` | 单镜头生成 | 15-30s | 1 GPU |
| `project_generate` | 整项目生成 | 5-30min | 多GPU |
| `lora_train` | LoRA模型训练 | 30-60min | 1 GPU |
| `feature_extract` | 角色特征提取 | 5-10s | CPU |
| `video_merge` | 视频合并导出 | 2-5min | CPU |
| `tts_generate` | 语音合成 | 3-10s | CPU |

## 五、任务取消设计

### 5.1 取消流程

```
用户点击「取消」
    ↓
Client 发送 DELETE /api/tasks/{task_id}
    ↓
API 更新任务状态为「cancelling」
    ↓
Celery Worker 检测到取消信号（Redis标志位）
    ↓
Worker 停止当前子任务，清理临时文件
    ↓
更新任务状态为「cancelled」
    ↓
WebSocket 推送取消确认消息
```

### 5.2 取消响应

```json
{
  "task_id": "task_7a3b9c2d",
  "status": "cancelling",
  "message": "正在取消任务，请稍候..."
}
```

### 5.3 取消限制

- **running 阶段**：Worker正在执行，无法立即取消，等待当前子任务完成
- **queued 阶段**：可立即取消，任务不会执行
- **pending 阶段**：可立即取消，任务不会执行

## 七、错误处理与重试

### 7.1 自动重试机制

```python
@app.task(
    bind=True,
    max_retries=3,
    default_retry_delay=60,
    autoretry_for=(APIError, NetworkError),
    retry_backoff=True
)
def shot_generate(self, params):
    try:
        # 生成逻辑
        result = call_generation_api(params)
        return result
    except APIError as e:
        # 指数退避重试
        raise self.retry(exc=e, countdown=2 ** self.request.retries)
```

### 7.2 错误码定义

| 错误码 | 说明 | 可重试 | 处理建议 |
|--------|------|--------|----------|
| `CHARACTER_EXTRACTION_FAILED` | 角色特征提取失败 | ✅ | 检查参考图，重新上传 |
| `MODEL_SERVER_ERROR` | 模型服务异常 | ✅ | 等待后重试 |
| `RESOURCE_EXHAUSTED` | 资源耗尽（GPU满） | ✅ | 排队等待 |
| `INVALID_PARAMS` | 参数错误 | ❌ | 修正参数后重新创建 |
| `TIMEOUT` | 任务超时 | ✅ | 自动重试或手动重试 |
| `CANCELLED` | 用户取消 | ❌ | 任务已取消，无需处理 |
| `QUOTA_EXCEEDED` | 额度用尽 | ❌ | 升级套餐或等待刷新 |

## 八、前端集成设计

### 8.1 WebSocket连接管理

```javascript
class TaskProgressManager {
  constructor(taskId) {
    this.taskId = taskId;
    this.ws = null;
    this.reconnectAttempts = 0;
    this.maxReconnectAttempts = 5;
  }

  connect() {
    this.ws = new WebSocket(`wss://api.example.com/ws/tasks/${this.taskId}`);

    this.ws.onmessage = (event) => {
      const data = JSON.parse(event.data);
      this.handleMessage(data);
    };

    this.ws.onclose = () => {
      this.handleDisconnect();
    };
  }

  handleMessage(data) {
    switch (data.type) {
      case 'progress':
        this.updateProgressBar(data.progress);
        this.updateStageInfo(data.stage_detail);
        break;
      case 'stage_change':
        this.showStageChangeToast(data.message);
        break;
      case 'success':
        this.handleSuccess(data.result);
        break;
      case 'failed':
        this.handleError(data.error);
        break;
    }
  }
}
```

### 9.2 WebSocket推送服务

```python
# websocket_manager.py
from fastapi import WebSocket
from typing import Dict
import asyncio

class TaskWebSocketManager:
    def __init__(self):
        self.active_connections: Dict[str, WebSocket] = {}

    async def connect(self, task_id: str, websocket: WebSocket):
        await websocket.accept()
        self.active_connections[task_id] = websocket

    def disconnect(self, task_id: str):
        if task_id in self.active_connections:
            del self.active_connections[task_id]

    async def send_progress(self, task_id: str, data: dict):
        if task_id in self.active_connections:
            await self.active_connections[task_id].send_json(data)

# Celery任务中推送进度
@app.task
def shot_generate(task_id: str, params: dict):
    for progress in generate_shots(params):
        # 推送到WebSocket
        send_task_progress.delay(task_id, {
            'type': 'progress',
            'progress': progress.percent,
            'stage_detail': progress.message
        })
```

### 9.3 Redis状态存储

```python
# task_status.py
import redis
import json
from enum import Enum

class TaskStatus(Enum):
    PENDING = 'pending'
    QUEUED = 'queued'
    RUNNING = 'running'
    SUCCESS = 'success'
    FAILED = 'failed'
    CANCELLED = 'cancelled'

def set_task_status(task_id: str, status: TaskStatus, data: dict):
    r = redis.Redis(host='localhost', db=0)
    key = f'task:{task_id}'
    r.hset(key, mapping={
        'status': status.value,
        'data': json.dumps(data),
        'updated_at': int(time.time())
    })
    r.expire(key, 86400)  # 24小时过期

def get_task_status(task_id: str) -> dict:
    r = redis.Redis(host='localhost', db=0)
    key = f'task:{task_id}'
    return r.hgetall(key)
```
