> 版本：v1.0 | 日期：2026-03-25 | 状态：待评审

## 一、背景与目标

### 背景
- MiniMax M2.7 高速版提供更快的推理速度和更好的中文理解能力
- 前端需要适配模型能力，提供流畅的AI生成体验
- 当前前端仅有login.html和register.html，核心编辑器缺失

### 目标
- 基于MiniMax M2.7模型实现故事→漫画的完整创作流程
- 前端使用Svelte框架，轻量快速
- 核心功能：故事输入 → AI生成剧本 → 分镜可视化 → 图片生成

## 三、页面结构

### 3.1 整体布局
```
┌──────────────────────────────────────────────┐
│  Header: Logo + 项目名 + 用户头像           │
├──────────────────────────────────────────────┤
│ Sidebar │ Main Content                      │
│ - 项目列表 │                                 │

│ - 历史记录 │                                 │
│           │                                 │
└──────────────────────────────────────────────┘
```

### 3.2 核心页面

#### 页面1：创作入口（/create）
- 故事输入框（textarea，500字以内）
- 风格选择：写实/动漫/水墨/厚涂
- 尺寸选择：16:9横版/9:16竖版/1:1方形
- 生成按钮："开始创作" → 调用MiniMax API生成剧本

#### 页面2：剧本编辑（/editor/:id）
- AI生成的剧本展示（可编辑）
- 角色列表（自动从剧本提取）
- 分镜列表（自动切分）
- 每个分镜可点击展开编辑

#### 页面3：分镜编辑（/shot/:shotId）
- 单个分镜的大图预览
- 角色描述编辑
- 对话/旁白输入
- 重新生成按钮

#### 页面4：作品预览（/preview/:id）
- 所有分镜漫画式展示
- 生成动态漫按钮
- 导出选项

## 五、API对接

### 5.1 现有后端接口（参考routes.py）
```
POST /api/story/generate     # AI生成剧本（使用MiniMax）
POST /api/shot/generate      # 生成分镜图片
GET  /api/project/:id       # 获取项目
PUT  /api/project/:id       # 更新项目
```

### 5.2 MiniMax M2.7调用
```javascript
// storyGenerate.js
export async function generateStory(text, style, ratio) {
  const response = await fetch('/api/story/generate', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      prompt: text,
      style: style,
      ratio: ratio,
      model: 'MiniMax-M2.7-highspeed'
    })
  });
  return response.json();
}
```
