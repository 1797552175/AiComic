## 一、项目概览

**项目名称：** AiComic · AI 创作动态漫平台

**一句话描述：** 从文字剧本到动态漫视频的一站式创作平台。用户输入剧本文字，系统自动完成分镜生成、画面绘制、动态效果添加、配音配乐，最终合成动态漫视频。

**当前代码进度：**
- ✅ 后端：完整实现，代码可直接运行
- ⚠️ 前端：仅占位文档，尚未开发

## 四、后端模块详解

### 4.1 配置层 — `config/config.py`

使用 Pydantic Settings 管理所有配置，读取 `.env` 文件：

```python
# 核心配置项
database_url     # PostgreSQL 连接字符串
redis_url       # Redis 连接
openai_api_key  # LLM API Key
llm_model       # LLM 模型名（默认 gpt-4）
sd_api_key      # Stable Diffusion API Key
s3_*            # 对象存储配置
celery_broker_url  # Celery 消息队列
```

### 4.2 数据模型 — `models/database.py`

5 张核心表，层级结构：

```
Project（项目）
  └── Character（角色）— 参考图、LoRA 路径、特征向量
  └── Scene（场景）— 场景地点/时间
        └── Shot（镜头）— 类型、时长、画面URL、运动参数
              └── Dialogue（对话）— 角色台词、情绪、音频URL、口型数据
```

**数据流向：** `Project → Scene → Shot → Dialogue`（通过外键关联，CASCADE 删除）

### 4.3 API 路由 — `api/routes.py`

REST 风格，按资源分组：

| 端点 | 方法 | 功能 |
|------|------|------|
| `/projects` | POST | 创建项目 |
| `/projects/{id}` | GET/PUT | 查询/更新项目 |
| `/projects/{id}/characters` | POST/GET | 角色管理 |
| `/projects/{id}/parse-script` | POST | **解析剧本**（核心入口）|
| `/projects/{id}/storyboard` | GET | 获取分镜列表 |
| `/projects/{id}/shots/{shot_id}` | PUT | 更新镜头 |
| `/projects/{id}/shots/generate-batch` | POST | 批量生成画面 |
| `/projects/{id}/shots/{shot_id}/apply-motion` | POST | 应用动态效果 |
| `/projects/{id}/compose` | POST | 合成最终视频 |
| `/options/motion-types` | GET | 获取运动类型列表 |
| `/options/shot-types` | GET | 获取镜头类型列表 |

**注意：** 当前 `generate-batch` 和 `compose` 接口返回 `task_placeholder`，实际异步任务队列（Celery）**尚未接入**，属于 TODO 项。

### 4.4 服务层 — `services/`

#### `script_parser.py` — 剧本解析
- 输入：原始剧本文本（支持【场景】标记和角色+情绪格式）
- 输出：结构化 JSON（scenes、characters、warnings）
- 实现：正则预处理 + LLM API 调用（GPT-4 / GLM-4）
- 风格候选：anime / realistic / cyberpunk / ink / bw

#### `storyboard.py` — 分镜生成
- 将剧本 Scene 细化为具体 Shot 列表
- 为每个 Shot 补充：type（镜头类型）、duration、keywords、description
- 镜头类型：wide / full / medium / close_up / two_shot / over_shoulder / pov

#### `image_generator.py` — 图像生成
- 调用 SDXL / FLUX 生成漫画画面
- 质量预设：standard (1024×768) / hd (1024×1024) / uhd (2048×2048)
- 风格预设：anime / realistic / cyberpunk / ink / bw
- 支持角色特征向量注入（LoRA/IP-Adapter）

#### `motion_engine.py` — 动态化引擎
- 镜头运动：static / pan_left / pan_right / tilt_up / tilt_down / zoom_in / zoom_out
- 口型同步（预留接口）：Wav2Lip / SadTalker
- 特效叠加：speed_line / rain / 等

#### `audio_service.py` — 音频服务
- TTS 合成：支持 Azure / CosyVoice / GPT-SoVITS / OpenAI
- 预设音色：xiaoming、xiaohong 等角色音色映射
- 情绪参数：pitch / speed 调节
- BGM 推荐：根据场景类型（happy / tense / romantic / sad）推荐配乐

#### `video_compositor.py` — 视频合成
- 输出质量：480p / 720p / 1080p / 4k
- 导出格式：mp4 / h265 / gif / webm / frames
- 合成流程：镜头动态化 → 片段合并 → 音频混流 → 编码输出

## 五、核心数据流

```
用户提交剧本文本
      │
      ▼
┌─────────────────┐
│  parse-script  │ ← script_parser.py（LLM 解析）
└───────┬─────────┘
        │ 写入 DB：Scene / Shot / Dialogue
        ▼
┌─────────────────┐
│  获取 storyboard │ ← 用户查看/编辑分镜
└───────┬─────────┘
        │
        ▼
┌─────────────────┐
│ generate-batch  │ ← image_generator.py（SD/FLUX 生图）
└───────┬─────────┘
        │ 更新 Shot.image_url
        ▼
┌─────────────────┐
│  apply-motion   │ ← motion_engine.py（镜头运动）
└───────┬─────────┘
        │ 更新 Shot.motion_data
        ▼
┌─────────────────┐
│    compose      │ ← video_compositor.py（FFmpeg 合成）
└─────────────────┘
        │
        ▼
     最终视频
```

## 九、架构约束与开发建议

1. **前后端分离**：前端通过 `/docs` 的 Swagger UI 或 REST API 与后端交互
2. **服务单一职责**：每个 `service` 文件对应一个领域，避免耦合
3. **异步优先**：全部使用 `async/await`，避免同步阻塞
4. **配置外置**：所有 secrets 和可变配置通过 `.env` 注入，不硬编码
5. **数据库迁移**：后续使用 Alembic 管理 schema 变更
6. **前端优先级**：当前最大瓶颈是前端未开发，建议优先推进
