# PM工作流程阻塞分析报告

> 产品经理机器人 | 日期：2026-03-24

---

## 一、发现的阻塞问题

### 阻塞1（致命）：所有Bot在02:05集体宕机

**证据**：`watchdog.log` 显示：
```
02:05:01 monitor (port 8001) DOWN - 重启失败
02:05:05 pm (port 8002) 重启失败
02:05:09 dev (port 8003) 重启失败
02:05:13 marketing (port 8004) 重启失败
```

**影响**：整个系统停摆，所有Bot不再响应。02:05之后几乎没有有效任务记录。

**根因**：可能是内存不足、端口冲突、或某个Bot的HTTP stub崩溃导致连锁反应。

---

### 阻塞2（致命）：monitor_cron.sh 脚本不存在

**证据**：`cron.log` 满屏都是：
```
/bin/bash: /opt/AiComic/状态报告/monitor_cron.sh: No such file or directory
```

**影响**：自动任务分发机制完全失效，每5分钟报错一次。Bot处于"不知道要干什么"的状态。

**根因**：cron job指向的脚本从未被创建。

---

### 阻塞3（严重）：PM→研发交接断环

**证据**：
- 22个原型在 `/opt/AiComic/原型/` 堆积，无一转化为研发任务
- 直到今天（03-24 09:00）才由用户推动手动创建了19个PROTO任务

**根因**：PM完成原型后没有自动通知研发，也没有强制创建Bitable任务。靠人工推动，靠用户提醒。

---

### 阻塞4（中等）：任务板状态失真

**证据**：T003在Bitable里标"进行中"，但实际上早已完成。研发BOT在03-23产出大量任务但没有更新Bitable状态。

**根因**：没有机制要求研发在完成commit后必须同步更新Bitable。commit和任务板是割裂的。

---

### 阻塞5（中等）：git commit没有强制规范

**证据**：cron.log反复显示`[Git] 有未提交的更改`，但从未自动提交。

**根因**：没有git hook或CI强制要求提交前更新文档。代码和文档脱节。

---

## 二、阻塞成因图

```
Bot宕机(02:05)
    ↓
所有Bot停摆（无法自动恢复）
    ↓
cron脚本缺失 → 任务分发中断
    ↓
PM不知道该干什么（等待）
    ↓
原型堆积，研发没有新任务
    ↓
两个Bot空闲 → 用户感知到"不干活"
```

## 三、规避机制设计

### 机制1：Bot健康检测 + 自动恢复（已部分实现）

**现状**：watchdog每5分钟检测一次，失败后尝试重启
**问题**：重启也失败时无进一步动作

**改进**：
- watchdog应实现"三级重启"：本地重启 → 清理端口 → 服务重启
- 重启失败时发送飞书通知给人工
- 增加内存监控（`free -m`），低于阈值不启动新Bot

---

### 机制2：重建monitor_cron.sh（紧急修复）

**当前**：脚本不存在，cron job每次报错

**需要**：
1. 创建 `/opt/AiComic/状态报告/monitor_cron.sh`
2. 功能：扫描 `原型/` 目录，发现新原型 → 自动创建Bitable任务 → @研发机器人
3. 每5分钟执行

---

### 机制3：PM→研发强制交接（流程约束）

**问题**：PM产出原型后没有强制动作

**改进**：
- **SKILL约束**：PM每输出一个原型，必须同时：
  1. 在功能追踪表（VTbwbtCgGaXKLCsF8BBc533OnAg）创建记录
  2. 在Agent任务板（InUZbPrTZaRm5LsRz9jctF27nGu）创建研发任务
  3. 在飞书群 @研发机器人 通知
- **禁止**："只输出原型文件"作为结束动作

---

### 机制4：研发完成后强制同步（git hook）

**问题**：研发commit代码后不更新Bitable

**改进**：
- 在 `.git/hooks/post-commit` 添加hook
- commit时扫描改动，如果是功能代码，提示更新Bitable状态
- 或在 `研发机器人` 的SKILL中强制：每完成一个commit，必须更新Bitable对应任务状态

---

### 机制5：每日待办自动推送

**问题**：Bot空闲时不知道该做什么

**改进**：
- 每天09:00，PM主动查询Bitable中"待领取"任务
- 如果有原型未创建研发任务，立即补充
- 如果没有待处理任务，执行竞品扫描自维护
