## 四、闭环机制设计

### 问题根因
产出的原型没有转化为研发任务录入任务板，导致流程断环。

### 闭环机制（新建）

```
原型产出 → PM评审 → 录入Bitable任务 → 研发领取 → 开发 → 验收 → 关闭
```

**新增字段建议（角色配置表）：**
- 原型路径
- 功能分类（P0/P1/P2）
- 依赖前置功能
- 研发状态（待领取/开发中/已完成/验证失败）

### 预防措施

1. **每次原型产出后**：自动创建对应TODO任务到Bitable
2. **T003任务**：更新状态为已完成
3. **定期巡检**：每周由PM巡检原型→任务转化率
4. **禁止空产出**：无实质内容的原型不允许提交
