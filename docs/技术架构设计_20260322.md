## 三、核心模块详细设计

### 3.1 剧本解析服务 (Script Parser)

**职责：** 将用户输入的剧本文本解析为结构化JSON

**输入：**
```
【场景1：街道·白天】
小明（高兴）：看，那家店新开业了！
小红（疑惑）：你怎么知道的？
```

**输出：**
```json
{
  "scenes": [{
    "id": "scene_1",
    "location": "街道·白天",
    "shots": [{
      "id": "shot_1",
      "type": "medium",
      "duration": 3,
      "keywords": "街道 白天 城市 远景",
      "dialogue": [{
        "character": "小明",
        "text": "看，那家店新开业了！",
        "emotion": "happy"
      }]
    }]
  }],
  "characters": [{
    "name": "小明",
    "emotion_default": "neutral"
  }]
}
```

**技术实现：**
- 使用LLM API进行解析（GPT-4 / GLM-4）
- Prompt工程：设计专用的解析Prompt，保证输出格式稳定
- 正则预处理：初步提取场景、角色、对话格式
- 异常处理：格式不规范时返回部分结果 + warning列表

### 3.2 分镜生成服务 (Storyboard Generator)

**职责：** 将剧本结构进一步细化为每个镜头的详细描述

**核心逻辑：**
```
1. 读取剧本JSON
2. 对每个scene，调用LLM生成shot列表
3. 每个shot补充：
   - 镜头类型（远景/中景/特写等）
   - 画面关键词（用于生图）
   - 时长预估
4. 返回完整的分镜列表
```

### 3.6 视频合成服务 (Video Compositor)

**核心工具：FFmpeg + Python**

```python
def compose_video(
    shots: List[dict],        # 分镜列表
    audio_track: List[dict],   # 音频轨
    output_path: str,
    quality: str = "hd"
) -> str:
    """
    合成最终视频
    1. 每个shot: 静态图片 + 动态效果 → 生成视频片段
    2. 合并所有片段
    3. 混入音频轨
    4. 加上字幕/水印
    5. 编码输出
    """

    # 步骤1: 为每个shot生成视频片段
    clip_paths = []
    for i, shot in enumerate(shots):
        img = load_image(shot["image_url"])
        motion = shot["motion"]  # 如 {"type": "pan_left", "duration": 3}
        audio = shot.get("audio")  # 该镜头的配音

        clip_path = f"/tmp/clip_{i}.mp4"

        # 使用FFmpeg生成动态片段
        cmd = build_motion_command(img, motion, clip_path)
        run_ffmpeg(cmd)

        clip_paths.append(clip_path)

    # 步骤2: 合并所有片段
    concat_list = " | ".join([f"-i {p}" for p in clip_paths])
    merged_path = "/tmp/merged.mp4"
    run_ffmpeg(f"ffmpeg {concat_list} -filter_complex concat {merged_path}")

    # 步骤3: 混入音频
    final_path = output_path
    run_ffmpeg(f"ffmpeg -i {merged_path} -i {audio_track} -c:a aac {final_path}")

    return final_path
```

**FFmpeg动态效果命令示例（pan左移）：**


### 4.2 核心表DDL

```sql
CREATE TABLE projects (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id VARCHAR(64) NOT NULL,
    title VARCHAR(255) NOT NULL,
    status VARCHAR(32) DEFAULT 'draft', -- draft/processing/completed
    settings JSONB, -- 风格、比例等全局设置
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

CREATE TABLE characters (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    project_id UUID REFERENCES projects(id) ON DELETE CASCADE,
    name VARCHAR(64) NOT NULL,
    description TEXT,
    lora_path VARCHAR(512), -- LoRA模型路径
    features_vector VECTOR(512), -- 角色特征向量 (pgvector)
    reference_images JSONB, -- 参考图URL列表
    created_at TIMESTAMP DEFAULT NOW()
);

CREATE TABLE shots (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    scene_id UUID REFERENCES scenes(id) ON DELETE CASCADE,
    order_index INT NOT NULL,
    type VARCHAR(32) DEFAULT 'medium', -- wide/medium/close_up/two_shot/over_shoulder/pov
    duration DECIMAL(4,2) DEFAULT 3.0, -- 秒
    keywords TEXT,
    image_url VARCHAR(512),
    motion_data JSONB, -- 运动参数 {"type": "pan_left", "duration": 3}
    status VARCHAR(32) DEFAULT 'pending', -- pending/generating/completed/failed
    retry_count INT DEFAULT 0,
    created_at TIMESTAMP DEFAULT NOW()
);

CREATE TABLE dialogues (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    shot_id UUID REFERENCES shots(id) ON DELETE CASCADE,
    character_name VARCHAR(64) NOT NULL,
    text TEXT NOT NULL,
    emotion VARCHAR(32), -- happy/sad/angry/surprised/neutral
    audio_url VARCHAR(512),
    lip_sync_data JSONB, -- 口型关键帧
    created_at TIMESTAMP DEFAULT NOW()
);
```

### 6.1 异步任务流

```
│  (Redis)   │ ──> Worker 2: 图像生成
│            │ ──> Worker 3: 音频生成
└──────┬──────┘
       │ 完成时推送WebSocket
       ▼
┌─────────────┐
│  前端更新   │
└─────────────┘
```

### 6.2 Celery任务示例

```python
from celery import Celery

app = Celery('aicomic', broker='redis://localhost:6379/0')

@app.task(bind=True, max_retries=3)
def generate_shot_image(self, shot_id: str, prompt: str, style: str):
    try:
        # 1. 调用SD生成图片
        image_url = sd_service.generate(prompt, style)

        # 2. 如有角色特征，应用LoRA
        if character_features:
            image_url = apply_character_lora(image_url, character_features)

        # 3. 更新数据库
        Shot.objects.filter(id=shot_id).update(
            image_url=image_url,
            status='completed'
        )

        # 4. 推送WebSocket进度
        send_progress(shot_id, 1.0, 'completed')

        return {"shot_id": shot_id, "image_url": image_url}

    except Exception as e:
        self.retry(exc=e, countdown=60)

@app.task
def compose_final_video(project_id: str):
    # 合成所有镜头为最终视频
    project = Project.objects.get(id=project_id)
    output_path = video_compositor.compose(project)
    project.output_url = output_path
    project.status = 'completed'
    project.save()
```

## 九、技术风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| SD生成角色不一致 | 高 | 重点优化LoRA特征注入，保留手动重绘入口 |
| FFmpeg合成速度慢 | 中 | 使用GPU加速 (NVENC)，分片渲染 |
| LLM解析剧本不稳定 | 中 | 多次调用取最优 + 人工兜底 |
| 高并发时GPU不足 | 高 | 任务队列 + 自动扩容 + 降级到CPU |
| 内容安全审核 | 中 | 接入阿里云/腾讯云内容审核API |
