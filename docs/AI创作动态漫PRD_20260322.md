# AI创作动态漫 — 产品需求文档（PRD）

> 版本：v1.0 | 日期：2026-03-22 | 状态：初版待评审

---

## 一、产品背景与目标

### 1.1 产品背景

动态漫（Motion Comic）是一种介于静态漫画和传统动画之间的内容形态，通过镜头运动、角色微动效、配音配乐让漫画"动起来"。相比传统动画，制作成本低、周期短，同时保留了漫画的分镜美学。

AI 技术的成熟使得自动生成分镜、自动画图、自动配音成为可能，大幅降低了动态漫的制作门槛。

### 1.2 产品目标

**核心目标：** 打造一款**以漫画创作为逻辑原点**的AI动态漫创作工具，让用户通过文字剧本一键生成完整的动态漫视频。

**关键指标（KPI）：**
- 用户完成一个动态漫项目的平均时长 ≤ 15 分钟（从剧本到导出）
- 单镜头画面生成成功率 ≥ 90%
- 角色一致性得分 ≥ 85%（同一角色不同镜头的视觉一致性）

## 二、用户与场景

### 2.1 目标用户

| 用户类型 | 画像 | 核心诉求 |
|---------|------|---------|
| 漫画作者 | 有漫画IP资产，想拓展视频渠道 | 快速将已有漫画视频化 |
| 自媒体/KOL | 需要持续内容产出 | 批量生成，降低视频制作成本 |
| AI爱好者 | 喜欢创作故事，无绘画基础 | 纯文字生成完整动态漫 |
| 企业/品牌 | 市场推广、内容营销 | 快速制作品牌故事短片 |

### 2.2 典型使用场景

**场景A：小明想讲一个科幻故事**
> 小明在脑子里构思了一个科幻故事，但没有绘画能力。他打开产品，输入一段文字剧本，选择"赛博朋克"风格，5分钟后获得了一个完整的动态漫视频。

**场景B：漫画作者小李想拓展读者群**
> 小李有一部连载的少女漫画。他上传剧本，选择与漫画一致的画风，系统自动保持角色一致性，生成动画版漫画预告片，发布到各平台吸引新读者。

**场景C：品牌方快速制作推广内容**
> 某游戏品牌上线新版本，在产品经理输入剧情大纲后，系统自动生成了一个1分钟的动态漫宣传片，当天发布。

### 3.2 模块1：剧本输入（Script Input）

#### 功能概述
支持多种形式的故事/剧本输入，AI辅助润色和格式规范化。

#### 详细需求

**R-1.1 纯文本输入**
- 提供富文本输入框，支持多段落
- 识别对话格式（括号内为旁白/动作，括号前为角色名）
- 格式示例：
  ```
  【场景1：街道·白天】
  旁白：城市的街道永远那么喧嚣。
  小明（高兴）：看，那家店新开业了！
  小红（疑惑）：你怎么知道的？
  小明：直觉。
  小红：……你什么时候有直觉了？
  ```

**R-1.2 脚本文件导入**
- 支持上传 .txt、.docx、.pdf 格式
- 自动解析并规范化格式
- 批量导入多个文件

**R-1.3 AI辅助扩写**
- 用户输入简要剧情（50-200字）
- AI自动扩展为完整剧本（包含场景描述、对话、情绪提示）
- 用户可编辑AI生成结果

**R-1.4 风格参考**
- 用户可粘贴参考文本/链接，锁定语言风格和叙事节奏
- 适用于保持系列作品的风格一致性

**R-1.5 智能格式检测**
- 自动识别角色名称，建立角色库
- 检测重复角色在不同场景的描述一致性
- 标记疑似错别字和不通顺语句

---

### 3.3 模块2：分镜生成（Storyboard Generation）

#### 功能概述
AI自动将剧本拆解为多个镜头，生成可视化的分镜列表，支持拖拽调整。

#### 详细需求

**R-2.1 AI自动分镜**
- 输入剧本，输出分镜列表（每个镜头包含：场景描述、镜头类型、时长预估、画面关键词）
- 默认镜头时长：2-6秒/镜
- 支持多场景自动分段

**R-2.2 镜头类型选择**
- 可选：远景、全景、中景、特写、双人中景、过肩镜头、主观镜头
- AI默认根据场景内容自动选择，可手动修改

**R-2.3 分镜可视化看板**
- 类似视频编辑时间线，上下拖拽调整镜头顺序
- 每个镜头显示：缩略图预览（占位）、时长、镜头类型
- 支持添加/删除/复制镜头

**R-2.4 批量时长调整**
- 可单独调整某个镜头时长（1-15秒）
- 可选中多个镜头批量调整速度

**R-2.5 分镜描述编辑**
- 每个镜头的"画面关键词"可手动编辑
- 影响后续画面生成的提示词

---

### 3.4 模块3：画面生成（Image Generation）

#### 功能概述
根据分镜描述生成高质量漫画画面，支持多种风格，核心解决**角色一致性**问题。

#### 详细需求

**R-3.1 文生图（Text-to-Image）**
- 输入：画面描述关键词 + 风格选择
- 输出：漫画风格图片
- 支持分辨率：1024x1024、1024x768（16:9）、768x1024（9:16）、1024x576（ultrawide）

**R-3.2 图生图（Image-to-Image）**
- 上传参考图 + 描述词，生成相似风格的新图
- 用于保持背景/场景一致性

**R-3.3 局部重绘（Inpainting）**
- 画笔涂抹选择区域，输入新描述词
- 只重绘选中区域，保留其他部分
- 用于修复AI生成中的瑕疵

**R-3.4 角色一致性（Character Consistency）⭐核心能力**
- 用户上传角色参考图（正面/侧面/表情各一张）
- 系统提取角色特征向量
- 后续该角色所有镜头自动继承特征，保证视觉一致
- 支持同时维护多个角色档案

**R-3.5 漫画风格选项**
| 风格ID | 名称 | 适用题材 |
|--------|------|---------|
| anime | 日系动漫 | 少女漫、少年漫 |
| realistic | 写实风格 | 剧情、悬疑 |
| cyberpunk | 赛博朋克 | 科幻 |
| ink | 水墨风 | 国风、古风 |
| bw | 黑白漫 | 硬汉、犯罪 |

**R-3.6 批量生成与备选**
- 同一镜头可一次生成4张备选
- 用户选择最满意的一张，或重新生成
- 支持"对所有镜头应用相同参数"批量生成

**R-3.7 画面参数调节**
- 画面比例：16:9 / 4:3 / 9:16 / 1:1
- 画质等级：标准 / 高清 / 超清
- 光影风格：自然光 / 戏剧光 / 柔光
- 氛围：暖色调 / 冷色调 / 自定义

---

### 3.5 模块4：动态化（Motion & Animation）

#### 功能概述
将静态画面转化为动态漫，通过镜头运动和局部微动效实现"动起来"的效果，无需全动画制作。

#### 详细需求

**R-4.1 镜头运动（Camera Motion）**
预设运动模板，应用到指定镜头：

| 运动ID | 名称 | 效果描述 |
|--------|------|---------|
| static | 静止 | 无运动，默认选项 |
| pan_left | 左平推 | 画面从右向左缓慢移动 |
| pan_right | 右平推 | 画面从左向右缓慢移动 |
| tilt_up | 上摇 | 镜头向上摇动 |
| tilt_down | 下摇 | 镜头向下摇动 |
| zoom_in | 推进 | 焦点向前推进 |
| zoom_out | 拉远 | 焦点向后拉远 |
| rotate | 旋转 | 画面轻微旋转 |
| shake | 抖动 | 轻微手持抖动效果 |

**R-4.2 口型动画（Lip Sync）**
- 输入对话文本
- AI分析音节节奏
- 自动生成匹配的口型动画
- 支持语言：中文（普通话/粤语）、英语、日语

**R-4.3 角色微动效**
预设动效，可叠加到指定镜头的指定角色：

| 动效ID | 名称 | 适用场景 |
|--------|------|---------|
| blink | 眨眼 | 静止说话镜头 |
| smile | 微笑 | 高兴场景 |
| breath | 呼吸起伏 | 静止对话 |
| hair_wind | 头发飘动 | 户外/风吹场景 |
| gesture | 手势 | 强调表达 |

**R-4.4 场景特效叠加**
| 特效ID | 名称 | 参数 |
|--------|------|------|
| speed_line | 速度线 | 方向/密度 |
| wind | 气流/风 | 方向/强度 |
| spark | 火花/光效 | 颜色/密度 |
| petal | 花瓣/落叶 | 类型/密度 |
| rain | 雨/雪 | 类型/强度 |
| blackout | 黑场过渡 | 时长 |
| whiteout | 白场过渡 | 时长 |

**R-4.5 动态效果批量应用**
- 可选中多个镜头，批量应用相同的镜头运动
- 预设组合：对话场景（静态+口型）、动作场景（推进+速度线）

---

### 3.6 模块5：配音配乐（Audio）

#### 功能概述
TTS文字转语音自动配音，智能推荐背景音乐，支持上传自定义音频。

#### 详细需求

**R-5.1 TTS语音合成**
- 每个角色可分配不同音色（预设音色库 ≥ 10种）
- 情感调节：高兴 / 悲伤 / 愤怒 / 平静 / 惊讶
- 语速调节：0.5x ~ 2.0x
- 音调调节：低沉 / 正常 / 高亢

**R-5.2 多语言混合**
- 一部作品中可有多语种对话
- 系统自动识别语言并匹配对应TTS模型

**R-5.3 背景音乐（BGM）**
- 内置BGM库，分类：欢快 / 紧张 / 温情 / 悬疑 / 战斗 / 悲伤
- AI根据场景情绪自动推荐BGM
- 支持MP3/WAV格式自定义上传
- 音量分别调节：BGM音量 / 人声音量
- 淡入淡出：镜头切换时自动过渡（默认1秒）

**R-5.4 音效（Sound Effects）**
- 内置音效库：门铃、脚步声、爆炸声、剑击声、手机铃声、汽车声、掌声、雨声等 ≥ 20种
- 用户可选择为特定镜头添加音效

**R-5.5 配音上传**
- 支持上传真实人声录音（MP3/WAV）
- 自动对齐台词与画面时间轴

---

### 3.7 模块6：合成输出（Export）

#### 功能概述
将所有元素（画面+动态+音频）合成为最终视频，支持多种格式和平台适配。

#### 详细需求

**R-6.1 输出格式**
| 格式 | 分辨率 | 用途 |
|------|--------|------|
| MP4 (H.264) | 1080P / 720P / 480P | 通用分发 |
| MP4 (H.265/HEVC) | 4K / 1080P | 高质量存档 |
| GIF | 原分辨率 | 社交媒体预览 |
| 序列帧 (PNG) | 原分辨率 | 后期制作 |

**R-6.2 封面生成**
- 自动截取精彩画面作为封面
- AI生成封面：输入标题自动设计
- 支持自定义上传封面图

**R-6.3 字幕设置**
- 字体选择：系统字体 / 手写体 / 漫画字体
- 颜色：白色 / 黑色 / 自动识别背景色
- 描边：無 / 细 / 粗
- 位置：底部 / 顶部 / 对话下方
- 背景：無 / 半透明黑框

**R-6.4 水印设置**
- 内置水印开关（水印内容：产品名称/LOGO）
- 自定义水印：位置（左上/右上/左下/右下）+ 透明度
- 防盗版选项：动态水印（画面移动时水印跟随移动）

**R-6.5 多平台尺寸适配**
- 一键导出多个尺寸版本：
  - 16:9 (YouTube/B站)
  - 9:16 (抖音/快手/小红书)
  - 1:1 (Instagram)

## 五、竞品对比与差异化

### 5.1 竞品表格

| 产品 | 类型 | 动态漫支持 | 角色一致性 | 分镜控制 | 门槛 |
|------|------|-----------|-----------|---------|------|
| Runway Gen-3 | AI视频 | ⭐⭐⭐⭐ | ⭐⭐ | ⭐ | 高 |
| Pika Labs | AI视频 | ⭐⭐⭐⭐ | ⭐⭐ | ⭐⭐ | 中 |
| 剪映 | 视频编辑 | ⭐⭐ | ❌ | ⭐⭐⭐⭐ | 低 |
| 腾讯智影 | AI视频 | ⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐ | 中 |
| **本产品** | AI动态漫 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | 低 |

### 5.2 差异化优势

1. **漫画原生逻辑**：从漫画创作流程出发（剧本→分镜→画面），而非视频创作逻辑
2. **角色一致性**：解决AI生成中角色"变脸"的痛点（核心技术壁垒）
3. **精细分镜控制**：让用户控制每个镜头的表达，而非AI随机生成
4. **动态漫专属动效**：专为"微动效"设计的镜头运动系统，不过度动画化
5. **一键多平台**：输出适配主流平台的不同尺寸版本

## 七、风险与依赖

| 风险/依赖 | 描述 | 应对措施 |
|-----------|------|---------|
| AI生图质量不稳定 | 角色一致性可能不达标 | 重点优化角色特征提取算法，保留人工重绘入口 |
| TTS情感表达 | 合成语音可能不够自然 | 引入多情感TTS模型，提供上传真人人声选项 |
| 内容审核误杀 | 审核系统可能误判正常内容 | 设置申诉通道，人工复审机制 |
| 视频合成性能 | 4K输出可能超时 | 分级渲染，先出预览再出高清 |

## 八、附录

### 8.1 术语表

| 术语 | 定义 |
|------|------|
| 动态漫 (Motion Comic) | 介于静态漫画和传统动画之间的内容形态，通过镜头运动和微动效让漫画"动起来" |
| 分镜 (Storyboard) | 将剧本拆解为连续镜头序列，每个镜头包含画面描述、时长、镜头类型等信息 |
| 角色一致性 (Character Consistency) | 同一角色在不同镜头中保持视觉特征一致的能力 |
| 镜头运动 (Camera Motion) | 画面在播放时的动态变化，如推进、摇动、旋转等 |
| TTS (Text-to-Speech) | 文字转语音技术 |

### 8.2 数据格式

**剧本输入格式（内部JSON）：**
```json
{
  "title": "故事标题",
  "scenes": [
    {
      "id": "scene_1",
      "location": "街道·白天",
      "shots": [
        {
          "id": "shot_1",
          "type": "medium",
          "duration": 3,
          "description": "城市街道的远景",
          "keywords": "城市街道 白天 远景 喧嚣",
          "dialogue": [],
          "image_prompt": ""
        },
        {
          "id": "shot_2",
          "type": "close_up",
          "duration": 2,
          "description": "小明高兴的表情",
          "keywords": "小明 高兴 表情 特写",
          "dialogue": [{"character": "小明", "text": "看，那家店新开业了！"}],
          "image_prompt": ""
        }
      ]
    }
  ],
  "characters": [
    {"name": "小明", "description": "阳光男孩", "reference_images": []},
    {"name": "小红", "description": "理性女孩", "reference_images": []}
  ]
}
```
