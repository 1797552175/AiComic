# PROJECT_SPEC.md - 产品规格与路线图

> 最后更新：2026-03-22

## 产品愿景

**一句话**：让漫画创作者通过 AI 将静态漫画一键动态化，生成可配音的动态漫视频。

**核心价值**：降低动态漫制作门槛，让创作者专注于创意本身。

---

## 核心功能

### P0 - 必须有（MVP）

| 功能 | 描述 | 状态 |
|------|------|------|
| 剧本输入 | 用户输入漫画脚本/对白 | 🚧 进行中 |
| AI 分镜 | 将剧本拆分为画面描述 | 🚧 进行中 |
| 画面生成 | 调用图像生成模型生成漫画分镜 | 🚧 进行中 |
| 动态化 | 让静态画面产生动效 | ⏳ 待开始 |
| 配音配乐 | AI 生成配音和背景音乐 | ⏳ 待开始 |
| 视频合成 | 将所有元素合成为最终视频 | ⏳ 待开始 |

### P1 - 下一版本

- 多角色协同（不同 Agent 负责不同模块）
- 风格一致性控制（保持画风统一）
- 用户自定义画风模板

### P2 - 规划中

- 社区分享（作品展示、点赞、评论）
- 创作者经济（付费解锁高级风格）
- API 开放（供其他平台接入）

## 技术架构

### Server A（OpenClaw · 飞书 Bot 协作层）

```
用户 → 状态监控机器人（协调）
           ↓
    产品经理 → 原型/PRD → 写入 docs/
           ↓
    研发机器人 → CrewAI 脚本 → SSH → Server B
           ↓
Server B（CrewAI + Claude Code 执行）
           ↓
    视频生成服务 → 输出到 /opt/AiComic/代码/output/
```

### Server B（执行层）

- Docker 容器：`crewai-runtime`
- 脚本路径：`/opt/AiComic/scripts/`
- 输出路径：`/opt/AiComic/代码/output/`
- 日志路径：`/opt/AiComic/logs/`
